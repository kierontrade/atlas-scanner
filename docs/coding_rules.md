# Coding Rules

- Dosyaları her zaman tam ver.
- Patch kod verme.
- Async yapıyı bozma.
- Modüler mimariyi koru.
- Her modül tek sorumluluk taşısın.
- Performans öncelikli yaz.
- Kod okunabilir olsun.
- Magic number kullanma.
- Mevcut fonksiyon isimlerini gereksiz değiştirme.
- JSON çıktıları bozulmamalı.
